# TASKS.md - 任务管理规范

## 任务类型

### 开发类任务
- **负责人**: dev-开发者
- **预估时间**: 根据复杂度 5-60 分钟
- **示例**: 代码编写、调试、架构设计、技术分析

### 调度类任务
- **负责人**: main-调度者
- **预估时间**: 即时
- **示例**: 任务分配、协调、状态跟踪

## 任务确认规范

### 接收任务时
```
✅ 任务已接收，正在处理：[任务简述]
```

### 处理中（长任务）
```
⏳ 进度更新：[当前进展]
```

### 完成时
```
✅ 任务完成：[结果摘要]
```

### 遇到问题时
```
❌ 遇到问题：[问题描述]
需要：[需要的帮助或资源]
```

## 任务优先级

- **urgent**: 紧急任务，立即处理
- **high**: 高优先级，优先处理
- **normal**: 正常优先级，按顺序处理
- **low**: 低优先级，有空时处理

## 超时设置

- 简单任务: 5 分钟
- 中等任务: 15 分钟
- 复杂任务: 30-60 分钟

超时后 main 会提醒或重新评估。
